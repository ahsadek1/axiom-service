ARCHITECTURE OVERVIEW
================================================================================

Five-Layer Failover System:

  LAYER 5: Graceful Degradation Coordinator
  └─ Maps health → degradation level → position size
     Level 0=100%, 1=100%, 2=50%, 3=0%, 4=0%

  LAYER 4: Verification & Reconciliation
  └─ Ensures no duplicates, capital accurate after failover

  LAYER 3: Standby Deterministic Algorithm
  └─ Simplified trading when primary down
     Rules: no new entries, close losers >2%, take winners >5%

  LAYER 2: Failover Trigger
  └─ Monitors health and activates failover at thresholds
     30s down = ACTIVATE, 5min down = ESCALATE

  LAYER 1: Health Detection
  └─ Monitors all 10 subsystems every 10 seconds
     SQS, TCP, HTTP, External API checks

Monitoring Timeline:
  Time 0:00   Component DOWN
  Time 0:30   FAILOVER ACTIVATED (30s threshold)
  Time 5:00   ESCALATION ALERT (if still down)
  Time N:NN   PRIMARY RECOVERS
  Time N:NN   RECONCILIATION CHECK
  Time N:NN   FULL OPERATION RESTORED
